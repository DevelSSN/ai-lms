\chapter{Implementation}

\section{Technologies Used}
The suggested implementation will leverage an advanced AI stack consisting of \textbf{FastAPI}, a high-performance Python framework for building APIs; \textbf{LangGraph}, a language model orchestration tool; \textbf{Qdrant}, a high-performance vector database; \textbf{PostgreSQL}, a robust relational database for user metadata; and \textbf{Redis}, a high-performance key-value store for session caching.

\section{Details of Implementation}

\subsection{AI Orchestrator}
The AI Orchestrator module can be considered a state-driven workflow managed by \textbf{LangGraph}. The main purpose of this component is to control the execution flow by utilizing the global \texttt{AgentState}. This object contains the message history, user profiles, and other information needed by agents to complete their tasks.
\begin{itemize}
	\item \textbf{State Management:} Implements a \texttt{TypedDict} structure that guarantees the availability of required data (e.g., knowledge level, retrieved documents) in every agent.
	\item \textbf{Routing Logic:} Leverages conditional routing techniques, which dynamically adjust the workflow depending on the \texttt{current\_task} (e.g., switching to \texttt{question\_gen} for assessments).
	\item \textbf{Integration with External Systems:} Uses \textbf{Redis} to store session-related information for low-latency response generation.
\end{itemize}

\subsection{User Profiling Module}
The above module is implemented via the \texttt{profiling\_agent}, which offers a standardized model for the learner at all times.
\begin{itemize}
	\item \textbf{Dynamic Profiling:} It relies on function calling within LLMs to interpret interactions and change the \texttt{knowledge\_level}.
	\item \textbf{Persistence Mechanism:} The \texttt{user\_profiles} PostgreSQL table is updated with \texttt{UPSERT} statements for unified management.
\end{itemize}
